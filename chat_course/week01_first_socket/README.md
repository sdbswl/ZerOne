# Week 1 — 네트워크와 소켓: 두 프로그램이 처음 대화한다

한 대의 PC 안에서 **서버 ↔ 클라이언트**가 메시지를 주고받는, 가장 단순한 채팅입니다.
이 주차는 일부러 **클래스 없이 함수와 절차(순서)만으로** 작성했습니다.

## 파일
| 파일 | 설명 |
|------|------|
| `server.py` | 손님을 기다리는 쪽. **먼저 실행**한다. |
| `client.py` | 말을 거는 쪽. 서버가 켜진 뒤 실행한다. |
| `make_ppt.py` | 강의자료 PPT 생성 스크립트 (강사용) |
| `Week01_네트워크와_소켓.pptx` | 완성된 강의 슬라이드 (발표자 노트 포함) |

## 실행 방법
터미널 **2개**를 띄웁니다.

```bash
# 터미널 1 — 서버 먼저 켜기
python server.py

# 터미널 2 — 클라이언트 연결
python client.py
```

클라이언트에서 한 줄을 입력하면 서버 화면에 떠오르고, 서버가 답을 입력하면
클라이언트 화면에 떠오릅니다. 종료는 `Ctrl+C`.

> 순서가 중요합니다. 서버를 켜지 않고 클라이언트를 먼저 실행하면 "연결 거부"
> 오류가 납니다 — 이것도 정상적인 학습 포인트입니다.

## 핵심 개념
- **IP = 전화번호, 포트 = 내선번호, 소켓 = 수화기**
- 서버: `socket → bind → listen → accept`, 클라이언트: `socket → connect`
- 네트워크엔 글자가 아니라 **바이트**가 흐른다 → `encode()` / `decode()` 필요

## 실습 / 과제
1. **에코 변형** — 서버가 받은 문장을 대문자로 바꿔 되돌려보내기 (`server.py` 맨 아래 힌트 참고)
2. 닉네임을 입력받아 `닉네임: 메시지` 형태로 표시하기
3. (가능하면) 같은 와이파이의 친구 PC와 IP로 연결해 보기
